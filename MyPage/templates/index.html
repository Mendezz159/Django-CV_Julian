Proyectos

Gestor de registros de salones de la UMB: Durante el tercer semestre de la carrera, por parte del área administrativa de la Universidad Manuela Beltrán fue requerida una aplicación para gestionar la ocupación de salones y se solicitó al grupo de semillero, del cual yo hacia parte, desarrollar esta aplicación. La aplicación tenia como objetivo que el personal de la universidad pudiera ingresar informacion de los salones y editar el estado del salón a una hora concreta. Y que los cambios realizados quedasen guardados en la nube para acceder a ellos fácilmente.
El proyecto se llevo a cabo con Excel Office y las plataformas Power Apps y Power Automate por requerimiento explícito del solicitante. Tuve a mi cargo el área del backend, donde inicialmente reestructuré los datos y registros ya existentes provistos por el área administrativa en Excel Office, como base de datos, para posteriormente guardarlos en la nube. Codifiqué la conexión entre la base de datos en la nube y la aplicación de usuario usando el lenguaje Microsoft Power FX, para generar los procesos de solicitud y filtrado de datos en Power Automate. Decidí utilizar la arquitectura Separación por Capas debido a la cantidad masiva de registros que los horarios de la universidad maneja. Tambien se dividió la conexión programando micro-servicios para optimizar la velocidad de respuesta debido a la densidad de datos mencionada. Se logró desarrollar exitosamente un CRUD operable desde la aplicación con distintas vistas creando una distribución de servicios de consulta asíncronos en Power Automate en un tiempo estimado de 6 meses durante la jornada universitaria.

Proyecto de ERP. E-TradeTech: En colaboración con un equipo de estudiantes durante el séptimo semestre de mi carrera hicimos una propuesta de ERP de 5 módulos enfocada en la gestión de comercios de hardware. La aplicación debía registrar los inventarios de almacenes, las solicitudes de pedidos, las listas de proveedores, los contratos de trabajadores y los datos contables. Estos datos debían ser suministrados al sistema por los usuarios asi como mostrar dichos datos dependiendo del usuario que los consulta. Por parte del frontend el proyecto se hizo con JavaScript y HTML5, mientras que por parte del backend se utilizó Java y Microsoft SQL Server.
Dentro de mis funciones en el proyecto estuvo el planteamiento lógico de la interaccion entre módulos haciendo los mapas de secuencia y mapas de actividades. A mi cargo estuvo el modelado completo de la base de datos asi como del montaje del entorno en los distintos equipos de prueba. Respecto al desarrollo, me encargué de desarrollar el backend codificando la conexión entre la base de datos SQL con el proyecto en Java, y usando esta conexión para crear un CRUD que pudiese controlarse desde el proyecto de java.
En cooperación con la otra parte del equipo encargada del frontend y la conexión de front a back, logramos programar las funcionalidades clave de cada uno de los 5 módulos, asi como crear su documentación en 4 meses durante la jornada universitaria.
